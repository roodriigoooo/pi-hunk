# 0.8.0

The review loop is now an explicit, changeset-aware checkpoint workflow.

## Added

- Complete changeset fingerprints compare live Hunk exports first and use a
  conservative Git working-tree fallback for pi-hunk-owned default reviews.
- Git freshness includes tracked edits, renames, untracked files, and
  untracked-file deletion.
- Checkpoint journal events are version 2 and persist the Hunk baseline,
  optional Git fallback, and render-record revision. Version-1 entries remain
  readable.
- `/hunk review` and `Ctrl+Shift+H` share one serialized action.
- Same-terminal review exits offer `Submit now`, `Keep for later`, and
  `Abandon`. Escape keeps the checkpoint for later.
- Persistent status and single-action inline hints expose ready, reviewing,
  re-review, approved, side-pane, and unknown-freshness states.

## Changed

- Freshness is reconciled at agent settlement, before review/submit/status,
  and during session restoration. Edit and write wrappers no longer invalidate
  checkpoints directly.
- A definite complete-changeset change persists `re_review_due`; reverting
  files later never restores approval automatically.
- Unknown freshness warns on submission but does not block unrelated Pi work.
- Hunk display titles are no longer treated as reviewed refs.
- The owned Hunk launch explicitly disables untracked-file exclusion.

## Migration

- The old automatic edit invalidation path is gone.
- The old `Resume` idea is not part of the workflow: same-terminal Hunk exits
  do not preserve an in-memory session that could be resumed.
- Legacy automatic pickup, agent-authored Hunk comments, sidecars, and deleted
  commands remain removed.

Publishing and tagging are intentionally outside this release change.
