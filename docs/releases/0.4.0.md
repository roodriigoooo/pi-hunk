# 0.4.0

Architectural seams so the renderer, review bridge, and configure UI share one
patch model instead of three parallel display models.

## Added

- **Shared patch-model seam.** The patch model now answers "which rendered hunk row does `(file, newLine)` land on" via `findPatchLineAddress`. Human review notes pin onto the exact changed row they refer to and render inline beside it (`│ note …`). `hunk_review_notes` returns a per-hunk shape matching the rendered surface, not a flat list. The renderer, `/hunk review`, auto-pickup correlation, and the review-tool result all cross one addressable-line query.
- **Highlighter cache module.** The lazy, keyed, single-flight, refresh-coalesced highlighter lifecycle moves out of `index.ts` closure soup into `src/highlighter-cache.ts`. `get(config)` serves the current highlighter (stale during a refresh); `refresh(config)` coalesces concurrent refreshes and notifies invalidates once the new key loads. Two interesting invariants — coalesce concurrent refreshes, serve stale until the new key loads — are now direct unit tests.
- **Config spec module.** The spec model (`ConfigSpec`, `ConfigGroup`, choice factories, color/symbol choices, descriptions, swatch) moves to `src/config-spec.ts` with a pure `specs → get/set/description/choices` interface. `configure.ts` keeps only TUI + orchestration and consumes the spec module. "Specs cover every config key", "every spec round-trips", and "every choice resolves" are now pure unit tests.

## Changed

- **Review views reuse the diff renderer.** `renderNotesLines` now renders the actual recent hunk with notes annotated on the addressed rows, instead of a flat text list. `renderReviewLines` groups notes by rendered hunk and marks each `✓ touched` or `○ open` using the shared address query.
- **File headers in unified patches** strip trailing `\t`-delimited labels (e.g. `before`/`after`) so review-note file correlation keys match the renderer.
