# 0.3.0

Diff rendering correctness, a reworked configure UI, and a sharper Hunk review loop.

## Fixed

- **Whole-hunk tokenization.** Each side of a hunk is now tokenized as one string so grammar state carries across lines. Multi-line constructs (template literals, block comments, JSX, triple-quoted strings) highlight correctly on continuation lines instead of being mis-tokenized as bare identifiers.
- **Word-emphasis indexing** verified for astral emoji and CJK: `diffWordsWithSpace` UTF-16-unit ranges stay aligned with the renderer's code-point cursor, so emphasis no longer bleeds past a 2-unit emoji onto the next character.

## Added

- **`/hunk review`** — a read-only TUI that pairs each open human note with whether a recent edit touched its line (`✓ touched`) or not (`○ open`), plus a summary count. Never sends anything to the agent.
- **Note-to-edit scoped auto-pickup.** `/hunk auto on` now injects only notes that overlap a recent edit by file and line. Notes about untouched files no longer trigger pickup on their own; arbitrary count thresholds no longer gate pickup.

## Changed

- **`/hunk configure` reworked.** Settings are grouped by what you see — *Side colors & words*, *Change rail*, *Header & line numbers*, *Symbols*, *Folding & limits*, *Shiki theme*, *Renderer & Hunk bridge* — behind a two-level nav. Esc inside a group returns to the group list; Esc on the group list saves and closes.
- **Color picker collapsed** from ~31 options per slot to 7 (`auto` + six UI-semantic colors) plus freeform `#RRGGBB`. The bare `green`/`red`/`gray` aliases and `theme:*` refs that duplicated `auto` or each other are removed from the picker; existing configs still resolve.

## Notes

- The grapheme-split edge case (combining marks, ZWJ emoji sequences where a diff boundary falls inside a visually-single character) is documented as rare and cosmetic, not fixed.
